# NHS Hospital Length of Stay (LOS) Prediction

A predictive analytics project for estimating hospital length of stay from surgical and patient data. Developed for **MN5812 Machine Learning & Predictive Analytics** (MSc Business Analytics). The pipeline loads and blends NHS-style data, engineers features, runs exploratory analysis, compares multiple regression models, and forecasts LOS for current in-patients.

---

## Overview

| Item | Description |
|------|-------------|
| **Target** | `LOS_Days` — continuous length of stay in fractional days |
| **Best model** | Linear Regression (RMSE ≈ 0.59 days, R² ≈ 0.78) |
| **Key predictor** | Hours to first ambulation (r ≈ 0.79 with LOS) |
| **Data** | 982 surgical records; patient, surgical, and ICD-10 lookup tables |

The workflow is implemented in **Python** (pandas, scikit-learn, matplotlib/seaborn) and optionally in **Alteryx** (`.yxmd`). Reports and charts use an NHS-inspired colour scheme.

## Quick start

### 1. Clone and set up environment

```bash
git clone https://github.com/YOUR_USERNAME/nhs-los-predictive-model.git
cd nhs-los-predictive-model
python -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Run the full pipeline

Place the four Excel files in the project root (or adjust paths in `analysis.py`), then:

```bash
python analysis.py
```

This will:

1. Load and merge **Patient Information**, **Surgical Information**, and **ICD-10 Codes**
2. Engineer features (age, hours to ambulation, surgery type, admission month/day, etc.)
3. Run EDA and save 10 charts to `charts/`
4. Train and compare five models (Linear Regression, Ridge, Decision Tree, Random Forest, Gradient Boosting)
5. Select the best model, save it as `best_model.pkl`, and produce evaluation plots
6. Score **Current Patients** and write `predictions_output.xlsx` (predicted LOS, discharge date, risk category)

### 3. Regenerate reports (optional)

```bash
python generate_report.py        # → 2024MN5812001.docx
python generate_deliverables.py  # → workflow + consultancy_report.docx
```

Update the constants at the top of these scripts if you have new metrics from a fresh `analysis.py` run.

## Data and features

- **Inputs:** Patient demographics, surgery timestamps, first ambulation time, ICD-10 procedure codes (with lookup for descriptions).
- **Target:** Length of stay in days (admission → release).
- **Features used in the model:** Age, hours to first ambulation, admission month, day of week, gender (one-hot), surgery type (one-hot from ICD-10 description). Numeric features are standardised before modelling.

Outliers in LOS are flagged (IQR rule) but not removed; the pipeline keeps all records for training and evaluation.

---

## Models compared

| Model | Notes |
|-------|--------|
| Linear Regression | Selected (best RMSE + CV R²). |
| Ridge Regression | α = 1.0. |
| Decision Tree | max_depth = 6. |
| Random Forest | 200 trees, max_depth = 10. |
| Gradient Boosting | 200 estimators, learning_rate = 0.05, max_depth = 4. |

Selection is by a simple rank sum of RMSE (lower is better) and 5-fold CV R² (higher is better). The chosen model is saved as `best_model.pkl`.

## Outputs

- **Charts** (`charts/`): LOS distribution, correlation heatmap, boxplots by surgery type/age/gender, LOS by ICD-10, procedure counts, ambulation vs LOS, feature importances, actual vs predicted, residuals, model comparison.
- **predictions_output.xlsx**: Patient ID, demographics, predicted LOS (days), predicted discharge date, risk category (Short / Medium / Long).
- **best_model.pkl**: Serialised scikit-learn model for reuse (e.g. in other scripts or Alteryx with Python SDK).
